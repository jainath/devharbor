(Unmodified builds straight from this repository may keep the DevHarbor name - this restriction
is about *modified* or *competing* distributions.)

The code is open so you can learn from it, audit it, run it, and improve it. The name and logo
identify the *official* project so users know what they're getting. Keeping those separate is a
common open-source practice (Firefox, WordPress, GitLab, and many others do the same).
